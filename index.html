在右键点击与拖动之间，开辟一条毫秒级的高效交互通道。

核心痛点

Windows 生态中，鼠标右键的能力被严重低估，用户与系统双双陷入了「单点即满足」的低效惯性。

🖱️

交互维度单一

数十年来，Windows 用户对右键的操作习惯被锁定在「单点打开菜单」。右键拖动这一天然、顺滑的交互维度，在系统层面长期缺位。

⏱️

效率存在盲区

快速启动软件、执行系统命令通常依赖键盘快捷键或层层点击。对于「手不离鼠」的用户，缺乏一种零记忆的快捷通道。

🎮

现有工具冲突多

市面上部分鼠标手势工具强行接管所有右键操作，导致在游戏、设计软件等专业场景中出现严重冲突，用户体验支离破碎。

核心功能

以像素级精度与系统级视野，重新定义鼠标右键的生产力边界。

01

智能识别单点与拖动

精准区分「右键单点」与「右键拖动」。若单点，正常弹出系统菜单，毫无违和感；若拖动，立即进入手势识别模式。

02

轨迹唤醒快捷方式

设定防误触像素阈值，支持 L型、Z型、直线下划、圆形等自定义轨迹。划出即唤醒，零延迟响应常用软件与系统命令。

03

全局覆盖与黑白名单

作为系统级工具全局运行，同时内置精细化的应用黑/白名单机制，彻底避免在游戏、渲染、设计等专业软件内发生冲突。

04

极低资源占用

底层采用原生 API 驱动，内存占用控制在个位数 MB。静默运行，不给系统增添一丝负担。

技术路线

以原生 Windows 技术栈为根基，追求极致性能与零侵入体验。

⚡

全局鼠标钩子 (SetWindowsHookEx)

基于 C# .NET / WPF 调用 Win32 API 注册底层鼠标钩子，捕获全局右键事件，确保在任何应用上层都能即时响应。

🔧

轻量化底层驱动

核心识别引擎采用向量化轨迹分析算法，路径采样、降噪、方向量化一气呵成，计算开销趋近于零。

🛡️

应用级白名单过滤

通过进程句柄比对与窗口焦点检测，毫秒级判定当前前台应用，自动禁用或启用钩子，保障游戏全屏与专业软件零冲突。

💾

本地配置持久化

手势与快捷方式映射采用 JSON 本地存储，启动时热加载，支持用户深度自定义与配置迁移。
